# Contributing Guide

Thank you for your interest in this project.

## Contribution policy

This repository is proprietary. Contributions are accepted only by explicit
authorization from the owner.

## Process

1. Open an issue describing the change.
2. Wait for maintainer approval before starting implementation.
3. Use a feature branch and open a pull request.
4. Keep scope small and include tests/build verification.

## Standards

- Keep all UI text in Spanish unless explicitly requested.
- Preserve offline-first behavior.
- Do not introduce external APIs without approval.
- Respect design system and contrast rules.

## Legal

By submitting a contribution, you confirm you have rights to the submitted
content and agree it can be used under this repository's proprietary terms.
